# Support

Superpowers Lite is maintained independently from upstream Superpowers. Choose
the destination that owns the behavior you observed.

## Report to Superpowers Lite

Open an issue at:

https://github.com/nanyumeng/superpowers-lite/issues

Use the Lite issue tracker when the problem reproduces with this repository and
concerns:

- a Lite skill rule or workflow decision;
- a Lite package, manifest, hook, or adapter;
- installation or upgrade instructions published by Lite;
- a regression between two Lite revisions;
- a matched upstream-versus-Lite behavior or token comparison.

Include the exact revision, model, effort, harness, installed plugins, minimal
reproduction, and a sanitized trace. For token reports, include correctness and
scope outcomes as well as the counting method.

## Report upstream

Use [obra/superpowers](https://github.com/obra/superpowers) when the issue also
reproduces on an unmodified upstream release and is unrelated to Lite's changed
policy. Search upstream open and closed issues before filing. Do not post a Lite
promotion or fork-specific request in an upstream issue.

## Report to the harness or model provider

Report to Codex, Claude Code, Cursor, Kimi, OpenCode, Pi, or another harness
when the failure persists without either Superpowers implementation, or when it
concerns account access, billing, model availability, sandbox enforcement, or
the harness's own tools.

## Security and conduct

Security vulnerabilities must use the private process in
[SECURITY.md](SECURITY.md), never a public issue.

During pre-release, a dedicated confidential Code of Conduct contact has not
yet been configured. Do not publish sensitive conduct details. Use the
maintainer's [GitHub profile](https://github.com/nanyumeng) to request a private
contact channel without describing the incident publicly. A dedicated private
address must be documented before the repository is announced as
community-ready.
