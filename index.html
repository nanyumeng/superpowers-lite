What changes

Less ceremony. Same guardrails.

Lite does not rebuild Superpowers or introduce another harness. It replaces the behavior-shaping skills inside one existing installation and leaves application projects, specs, plans, and adapter code untouched.

01

Risk-proportional flow

Clear documentation and configuration changes can proceed directly. Ambiguous features still get research, a specification, and a written plan.

02

Selective orchestration

Subagents, worktrees, systematic debugging, and review loops remain available, but trigger when isolation, complexity, or risk justifies them.

03

Engineering discipline

Production behavior changes still begin with a failing test. Completion still requires fresh evidence. Destructive actions still require confirmation.

Evidence, honestly

Measurements are separated from observations.

Before you install

One active adapter. One recoverable backup.

Keep a working upstream Superpowers installation so its native adapter can discover the skills. Back up its complete skills/ directory, replace that directory with Lite, verify both trees, and retain the backup for rollback.
