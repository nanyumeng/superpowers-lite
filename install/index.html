Safe installation

Replace one directory. Keep the way back.

Superpowers Lite is a skills overlay, not a standalone harness plugin. The safe path keeps one active upstream adapter, creates a unique backup, replaces the complete skills tree, and verifies both sides.

Before you start

You need one working obra/superpowers installation. Lite is an unofficial derivative based on upstream v6.1.1 and relies on that installation's native adapter for skill discovery and harness integration.

  • Do not enable two copies of the same superpowers:* skills.
  • Do not guess the active installation path. Let the coding agent inspect the harness metadata.
  • Do not delete the original skills tree. Move it to a unique, timestamped backup.
  • Install a pinned release when reproducibility matters: v6.1.1-lite.2.

Give this to your coding agent

The prompt asks the agent to complete the operation, surface ambiguous paths instead of guessing, and report an exact rollback command.

Install or update Superpowers Lite on this machine. Complete the installation; do not only explain the steps.

Source release: https://github.com/nanyumeng/superpowers-lite/releases/tag/v6.1.1-lite.2

1. Find the one active Superpowers installation and its exact skills/ path using this harness's plugin metadata or manager. If Superpowers is absent, install upstream Superpowers through the harness's native method first. If paths are ambiguous, stop and ask me.
2. Download the release to a temporary directory and validate skills/using-superpowers/SKILL.md. If its complete skills/ tree already matches the active one, report that no update is needed.
3. Show the active path and a unique timestamped Backup path, then move the complete active skills/ directory to that Backup. Do not delete, merge, or overwrite anything.
4. Copy the complete superpowers-lite skills/ directory into the original location. Preserve every file outside skills/, then verify the installed and downloaded trees match recursively and the Backup is intact.
5. Report the original version, installed path, Backup path, source commit, verification result, restart/new-session requirement, and an exact Rollback command. Do not execute Rollback.

Change no application project, project configuration, AGENTS.md, CLAUDE.md, or docs/superpowers/ content. Do not claim success without fresh verification.

Verify in a clean session

Restart the harness or open a new session. The UI may still say “Superpowers” because Lite intentionally keeps the installed adapter. In a disposable repository, test two different task shapes:

AmbiguousLet's make a React todo list. should trigger brainstorming before implementation.
ExplicitA small documentation correction should proceed directly and end with fresh verification.

This smoke test checks routing behavior. It does not prove token savings, accuracy, or compatibility with every harness.

Rollback

Use the exact backup path reported during installation. The rollback itself should remain recoverable and should not touch application projects.

Rollback Superpowers Lite using this exact Backup path:
<PASTE_BACKUP_PATH>

Find the currently active Superpowers skills/ directory, verify that the Backup contains using-superpowers/SKILL.md, show both exact paths, and restore the Backup using recoverable move operations. Do not delete either tree or modify project files. Verify the restored skills and tell me whether to restart the harness or open a new session.

When upstream updates

An update to the upstream plugin may replace the overlaid skills. Re-run the same installation process against the current Lite release, creating a new backup each time. Review the compatibility boundary and evidence limits before adopting Lite in a critical workflow.

Ownership and support

This guide and the overlay are maintained by nanyumeng. They are unofficial and are not endorsed or supported by Superpowers, OpenAI, or Anthropic. Report Lite-specific problems in the Lite issue tracker, not upstream issue threads.
