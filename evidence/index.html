Evidence and limitations

Know which numbers answer which question.

Superpowers Lite is an unofficial derivative of obra/superpowers v6.1.1. Its public numbers include source measurements, a maintainer observation, and a small directional test. They should not be blended into one performance claim.

  1. Source size
  2. Field observation
  3. Micro-test
  4. Not proven
  5. Evaluate it

Reproducible source measurement

3,322Lines across 14 core upstream SKILL.md files at v6.1.1 (d884ae0).
1,053Lines across the corresponding 14 Lite core SKILL.md files in the launch baseline.

The difference is 2,269 lines, or 68.3%. The complete skills layout remains 48 files. In the current Lite tree, 36 are Markdown and 12 are helper files used by the skills.

Interpretation

This establishes that the core instruction source is smaller. It does not establish a 68.3% reduction in processed tokens, total task cost, latency, errors, or tool calls.

Maintainer field observation

Maintainer field observation: in ongoing real-project use after switching from upstream Superpowers to Lite, the maintainer observed 30%+ lower token use, faster completion, and more focused first-pass results with less unnecessary development.

This is not a controlled benchmark. The work was not randomized or held constant across model, repository state, prompt, harness, cache, and task difficulty. It is useful motivation for further testing, not a universal prediction.

Earlier three-task directional micro-test

An earlier three-task comparison produced mixed results: processed tokens were 2.8% lower and fresh input was 22.2% lower for Lite. The test was small, used different work phases, and did not establish an accuracy advantage. Its benchmark artifact is not part of the current minimal distributable tree.

What is not proven

  • That Lite is always faster or cheaper.
  • That fewer instructions always improve results.
  • That Lite produces more accurate code than upstream.
  • That results from one model, harness, or repository generalize to another.
  • That the 30%+ field observation is caused solely by the overlay.
  • That vendor prompting guidance endorses this project.

Superpowers, OpenAI, and Anthropic have not evaluated or endorsed Lite.

How to evaluate it in your environment

  1. Choose representative tasks before switching.
  2. Pin the model, harness version, repository state, prompt, and tools.
  3. Run both conditions in fresh sessions and randomize their order.
  4. Judge task correctness and unnecessary changes blind when possible.
  5. Record processed tokens, fresh input, output, tool calls, latency, and corrections.
  6. Keep failures and neutral results, not only wins.

Use the reversible installation and Rollback guide, then share reproducible results in the Lite issue tracker. This page is maintained and promoted by nanyumeng.
